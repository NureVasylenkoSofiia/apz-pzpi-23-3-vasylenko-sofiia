3 ОПИС ВИКОНАНОЇ РОБОТИ


3.1 Загальні відомості про шаблони (патерни) проєктування
Патерни проєктування — це типові рішення поширених проблем проєктування програмного забезпечення. Вони не є готовим кодом, а представляють собою узагальнені підходи до організації взаємодії об'єктів.
Основне призначення патернів:
* підвищення повторного використання коду
* спрощення підтримки системи
* зменшення зв’язаності між компонентами
Відповідно до класифікації GoF (Gang of Four), патерни поділяються на:
* породжувальні (створення об’єктів)
* структурні (організація класів і об’єктів)
* поведінкові (взаємодія між об’єктами)
3.2 Опис обраного патерна
* Назва патерна: Factory Method
* Тип шаблона (патерна): породжувальний
* Основне призначення: патерн Factory Method визначає інтерфейс для створення об’єкта, але дозволяє підкласам змінювати тип створюваного об’єкта. Іншими словами, він делегує створення об’єктів дочірнім класам.
* Передумови застосування: Патерн доцільно використовувати, коли:
   * клас не знає наперед, об’єкти якого типу йому потрібно створювати
   *  необхідно делегувати створення об’єктів підкласам
   *  потрібно забезпечити слабке зв’язування між клієнтом і конкретними класами
* Переваги: 
   * зменшує залежність від конкретних класів
   * підтримує принцип відкритості/закритості (Open/Closed)
   * спрощує розширення системи
* Недоліки: 
   * збільшує кількість класів у системі
   * ускладнює структуру програми при неправильному використанні 
3.3 Структура шаблона (патерна)
* Опис структурних частин:
   * Product (Продукт)
Загальний інтерфейс або абстрактний клас для створюваних об’єктів
   * ConcreteProduct (Конкретний продукт)
Реалізація інтерфейсу Product
   * Creator (Творець)
Оголошує фабричний метод, який повертає об’єкт типу Product
   * ConcreteCreator (Конкретний творець)
Реалізує фабричний метод і створює конкретні об’єкти
   * Опис взаємодії об’єктів
   * Клієнт працює з Creator
   * Creator викликає factory method
   * ConcreteCreator повертає конкретний продукт
   * Клієнт не знає точного класу об’єкта


   * UML-діаграми з поясненнями

Приклади програмного коду


        Напиши, будь ласка, простий приклад реалізації патерна Factory Method на C#. Потрібно створити інтерфейс ITransport з методом Deliver(), а також класи Truck і Ship, які його реалізують. Ще потрібен абстрактний клас Logistics з фабричним методом CreateTransport() і методом, який створює об’єкт і викликає Deliver().
В.1 Приклад коду
1  public interface ITransport
2  {
3      void Deliver();
4  }
5
6  public class Truck : ITransport
7  {
8      public void Deliver() =>
9          Console.WriteLine("Доставка вантажівкою");
10  }
11
12 public class Ship : ITransport
13 {
14     public void Deliver() =>
15         Console.WriteLine("Доставка кораблем");
16 }
17
18 public abstract class Logistics
19 {
20     public abstract ITransport CreateTransport();
21
22     public void PlanDelivery()
23     {
24         var transport = CreateTransport();
25         transport.Deliver();
26     }
27 }




        Допоможи доповнити приклад патерна Factory Method на C#. Потрібно створити класи RoadLogistics і SeaLogistics, які будуть повертати відповідно Truck і Ship через фабричний метод. Також покажи приклад використання в методі Main().
В.2 Приклад коду
1  public class RoadLogistics : Logistics
2  {
3      public override ITransport CreateTransport()
4          => new Truck();
5  }
6
7  public class SeaLogistics : Logistics
8  {
9      public override ITransport CreateTransport()
10          => new Ship();
11 }
12
13 class Program
14 {
15     static void Main()
16     {
17         Logistics logistics = new RoadLogistics();
18         logistics.PlanDelivery();
19
20         logistics = new SeaLogistics();
21         logistics.PlanDelivery();
22     }
23 }
